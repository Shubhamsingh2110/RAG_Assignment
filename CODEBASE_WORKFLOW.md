# Berkshire Hathaway Intelligence - Codebase Workflow

## Goal
Build the assignment as a Mastra-based RAG application that answers questions about Warren Buffett and Berkshire Hathaway using the annual shareholder letters from 2019 to 2024.

This document is the working contract for the codebase. It freezes the stack, defines the implementation order, and explains how to move forward without changing the requested technology choices.

## Frozen Stack
Do not change these unless the user explicitly approves it later.

- Framework: Mastra
- Language: TypeScript
- LLM: OpenAI GPT-4o
- Embeddings: OpenAI text-embedding model
- Vector store: PostgreSQL with pgvector
- Core features: RAG, Agents, Workflows, Memory, Streaming
- Source material: Berkshire Hathaway shareholder letters in PDF form

## Non-Goals
- Do not replace Mastra with another framework.
- Do not swap the LLM provider.
- Do not switch away from PostgreSQL + pgvector unless required by Mastra setup.
- Do not introduce extra application frameworks just for convenience.
- Do not add product features that are unrelated to the assignment.

## Working Rules For This Repo
- Keep the implementation grounded in Mastra's documented patterns.
- Use the assignment PDFs as the only knowledge base for the assistant.
- Preserve source attribution for every answer.
- Keep all document processing, retrieval, and chat logic inside the codebase.
- Prefer simple, explicit modules over hidden magic.
- Build a working path first, then refine the UX.

## Suggested Repository Shape
The exact scaffold will depend on the Mastra CLI output, but the codebase should conceptually follow this structure:

- `src/`
- `src/agents/` for the Buffett agent
- `src/tools/` for retrieval and search tools
- `src/workflows/` for ingestion and ETL flows
- `src/rag/` for chunking, indexing, and retrieval helpers
- `src/memory/` for conversation memory setup
- `src/lib/` for shared Mastra clients and config
- `app/` or the Mastra-generated frontend entrypoints for streaming chat UI
- `data/source/` for the shareholder PDFs
- `data/processed/` for extracted text and chunk artifacts
- `docs/` for setup, testing, and deployment notes

## Implementation Order

### Phase 0 - Verify Inputs
1. Download or place the Berkshire Hathaway letters in `data/source/`.
2. Confirm the final Mastra scaffold generated by `create-mastra@latest`.
3. Set environment variables for OpenAI and PostgreSQL before any ingestion runs.

### Phase 1 - Project Bootstrap
1. Initialize the project with `npx create-mastra@latest`.
2. Select the Mastra options for RAG, Agents, and Workflows.
3. Keep the generated structure and only add code needed for this assignment.
4. Confirm the local dev workflow works before adding app logic.

### Phase 2 - Document Ingestion
1. Parse each PDF shareholder letter.
2. Convert each document into Mastra `MDocument`-compatible inputs.
3. Chunk the text with a strategy suitable for long-form financial letters.
4. Store document metadata such as year, title, and source file path.
5. Persist the chunks and embeddings in the vector database.

### Phase 3 - Retrieval Layer
1. Build a search tool that queries the vector store.
2. Support metadata filters by year and source document.
3. Return chunk text plus attribution metadata.
4. Keep retrieval output simple enough for the agent to use directly.

### Phase 4 - Agent Layer
1. Create one primary Berkshire Hathaway analyst agent.
2. Bind it to GPT-4o through Mastra's model configuration.
3. Add persistent conversation memory.
4. Make the system prompt enforce grounded answers, citations, and clear limitations.
5. Wire retrieval tools into the agent workflow.

### Phase 5 - Streaming Chat UI
1. Connect the frontend to the agent's streaming response path.
2. Render partial responses as they arrive.
3. Show cited sources alongside each answer.
4. Add loading, error, and empty states.
5. Keep the interface minimal and focused on question answering.

### Phase 6 - Validation
1. Test direct questions about Buffett's philosophy.
2. Test follow-up questions that depend on prior context.
3. Test year-specific questions about acquisitions or strategy changes.
4. Verify citations point to the correct shareholder letters.
5. Confirm the app still works after restart.

### Phase 7 - Documentation And Delivery
1. Document setup and environment variables.
2. Document how ingestion is run.
3. Document how to start the dev server and test the playground.
4. Document how citations and memory work.
5. Document deployment notes once the app is stable.

## Immediate Next Tasks
1. Run the Mastra CLI scaffold if it has not been created yet.
2. Add the Berkshire Hathaway PDFs to the source folder.
3. Create the ingestion workflow and vector indexing path.
4. Build the chat agent and streaming UI on top of that pipeline.
5. Add a short test checklist for the assignment queries.

## Acceptance Criteria
The codebase is ready when all of the following are true:

- The Mastra project boots locally.
- The letters are ingested and chunked successfully.
- The vector store returns relevant passages.
- The agent answers with source-backed citations.
- Follow-up questions keep conversation context.
- The chat UI streams responses in real time.
- The repository contains clear setup and testing notes.
